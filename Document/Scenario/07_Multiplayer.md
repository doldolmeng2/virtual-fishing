# 시나리오 07 - 멀티플레이 (점수 공유)

> **구현 우선순위: 낮음** — 핵심 기능 완성 후 시간이 남을 경우 구현 예정.
> 초기 버전에서는 **서로의 점수만 공유하여 보여주는 간단한 형식**으로만 지원합니다.

## 도메인

| 도메인 | 클래스 / 인터페이스 | 역할 |
|---|---|---|
| 플레이어 | `AccountDataSO` | 로컬 점수 데이터 관리 |
| 게임 서버 | `GameServerService` (미정) | 점수 송수신, 리더보드 관리 |
| 피드백 | `FeedbackManager` / `IFeedbackService` | 리더보드 UI 렌더링 |

## 사전 조건

1. 플레이어가 인터넷에 연결되어 게임 서버의 특정 방(또는 글로벌 로비)에 세션 연결
2. 게임 서버가 접속 중인 플레이어들의 점수(`totalScore`) 목록을 유지 중
3. 플레이어가 시나리오 05(포획 결과)에서 물고기 포획을 성공적으로 완수한 직후

## 로직

### 1단계: 점수 계산

1. 클라이언트 내부에서 획득한 물고기의 속성(크기, 희귀도 등)을 바탕으로 이번 턴 점수 계산
2. `AccountDataSO.totalScore`에 가산

### 2단계: 서버 동기화

1. 계산된 점수를 게임 서버로 전송 (API 호출 또는 소켓 메시지)
2. 게임 서버가 해당 플레이어의 총점을 갱신하고 리더보드를 정렬
3. 서버가 갱신된 리더보드 데이터를 모든 연결된 클라이언트에 브로드캐스트

### 3단계: 리더보드 표시

1. 클라이언트의 `FeedbackManager`가 서버 응답을 파싱
2. VR 환경 내 전광판 또는 허공의 UI 캔버스에 리더보드를 크고 읽기 쉬운 텍스트로 표시
   - 순위, 이름, 점수
   - 본인 순위 하이라이트

## 속성 변화

| 속성 | 변화 내용 |
|---|---|
| `AccountDataSO.totalScore` | 획득한 물고기 가치만큼 증가 |
| 서버 `leaderboardList` | 갱신된 점수 반영 후 정렬 |
| 리더보드 UI | 최신 데이터 기반으로 텍스트 렌더링 |

## 사후 조건

1. 플레이어의 점수가 서버에 안전하게 저장 및 동기화됨
2. 플레이어가 VR 화면에서 갱신된 본인 및 타인의 점수를 확인 가능

## 비고

- 네트워크 장애 시 로컬에 점수를 캐싱하고, 연결 복구 시 동기화하는 방어 로직 필요
- 서버 아키텍처 및 프로토콜은 별도 설계 필요
- 이 시나리오의 인터페이스는 핵심 기능 완성 후 정의 예정
